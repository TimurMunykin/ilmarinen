# App Template

Boilerplate for apps generated by Ilmarinen platform.

## Placeholders

The following placeholders must be replaced by App Engine during scaffolding:

| Placeholder | Description | Example |
|-------------|-------------|---------|
| `{{APP_NAME}}` | Human-readable app name | `My Pet Tracker` |
| `{{APP_SUBDOMAIN}}` | URL-safe subdomain | `my-pet` |

## Generated Code Markers

The following markers indicate where AI-generated code is inserted:

| Marker | File | Purpose |
|--------|------|---------|
| `// === GENERATED MODELS BELOW === //` | `apps/api/prisma/schema.prisma` | Prisma models |
| `// === GENERATED MODULE IMPORTS === //` | `apps/api/src/app.module.ts` | NestJS import statements |
| `// === GENERATED MODULES === //` | `apps/api/src/app.module.ts` | NestJS module registrations |
| `// === GENERATED NOTIFICATION RULES === //` | `apps/api/src/modules/notifications/notifications.service.ts` | Notification rule definitions |

## Directory Structure for Generated Code

- `apps/api/src/modules/` — NestJS business modules (one dir per model)
- `apps/web/src/routes/_app/` — React pages (added as new files, no markers needed — TanStack Router auto-discovers route files)
- `apps/web/src/components/` — Custom React components (added as new files)
- `apps/web/src/lib/i18n/` — Translation files (App Engine merges generated keys into existing ru.json/en.json)

## Environment Variables

See `.env.example` for all required variables. App Engine injects these at deploy time.

## Auth

Auth is delegated to the Ilmarinen platform. The app validates JWTs using the platform's RSA public key. No Google OAuth credentials needed per app.

## Notifications

Notification rules are defined in `notifications.service.ts` as a typed array. The cron job runs daily at 9:00 AM and evaluates all rules against the database.

Supported condition formats:
- `daysUntil(<dateField>) <= N`
- `daysSince(<dateField>) >= N`
