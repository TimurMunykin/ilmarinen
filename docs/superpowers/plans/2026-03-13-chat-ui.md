# Chat UI Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a chat interface for creating apps via natural language — backend ChatModule drives OpenAI conversation, frontend renders chat with spec proposal cards.

**Architecture:** New NestJS ChatModule (service + controller) handles sessions and messages, calls OpenAI to form AppSpec. React frontend adds chat route with message list, input bar, and spec card component. On confirm, calls existing createApp + generateApp APIs.

**Tech Stack:** NestJS 11, OpenAI API (`openai` package), Prisma, React 19, TanStack Router, shadcn/ui, i18next

**Spec:** `docs/superpowers/specs/2026-03-13-chat-ui-design.md`

### Task 2: Chat service — session CRUD + AI conversation

**Files:**
- Create: `platform/apps/api/src/modules/chat/chat.service.ts`
- Create: `platform/apps/api/src/modules/chat/chat.service.test.ts`
- Create: `platform/apps/api/src/modules/chat/chat-prompts.ts`

**Chat service methods:**
- `createSession(userId: string)` → creates ChatSession, returns `{ id }`
- `getSession(sessionId: string, userId: string)` → returns session with messages, throws ForbiddenException if not owner
- `sendMessage(sessionId: string, userId: string, content: string)` → saves user message, calls OpenAI, saves AI response, returns AI message DTO

**`sendMessage` logic:**
1. Verify ownership, check message count ≤ 30 (throw BadRequestException if exceeded)
2. Save user ChatMessage (role: "user")
3. Load all session messages
4. Resolve API key via `AiAccessService.resolveApiKey(userId)`
5. Build messages array: system prompt (from `chat-prompts.ts`) + history (map role/content)
6. Call `openai.chat.completions.create` with model from `OPENAI_MODEL` config, `response_format: { type: 'json_object' }`
7. Parse JSON response `{ message, spec }` — save as ChatMessage (role: "assistant", content: message, metadata: `{ spec }` if spec non-null)
8. Return DTO with `{ id, role, content, metadata, createdAt }`

**`chat-prompts.ts`:** Export `buildChatSystemPrompt()` that returns the system prompt string. Must include the word "JSON" (required for json_object mode). Instructs AI to be a friendly app designer, ask questions, and produce AppSpec when ready. Reference the `AppSpec` interface shape (from `engine/prompts.ts`) so AI knows the exact structure.

**Tests (mock OpenAI + PrismaService + AiAccessService):**
- `createSession` — creates session for user
- `getSession` — returns session with messages; throws ForbiddenException for wrong user
- `sendMessage` — saves user message, calls OpenAI, saves AI response, returns DTO
- `sendMessage` — throws if no API key
- `sendMessage` — throws BadRequestException when message limit (30) exceeded
